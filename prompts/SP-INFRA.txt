# OpenHash — 계층적 분산원장 (특허 제10-2026-0018910호)

## 5계층 구조 (PLSM — 확률적 계층 선택)
| 계층 | 저장소 | 역할 | 비고 |
|---|---|---|---|
| L1 | openhash-L1-ido1 | Edge 노드 (읍면동) — 즉시 해시 등록 | 평균 **4ms** 이내 |
| L2 | openhash-L2-jeju-city | Regional (시군구) — 지역 집계·검증 | |
| L3 | openhash-L3-jeju | National (광역시도) — 국가 무결성 | |
| L4 | openhash-L4-kr | 국가 클러스터 — 최소 **13개** 물리 노드 | 검열 저항 자동 격리 |
| L5 | openhash-L5-global | 글로벌 — **19개 이상** 대표 노드 | 영구 보존·법원 등록 |

## 핵심 기술 (특허 v4 명세)
- **PLSM**: 확률적 계층 선택 (이중 SHA-256 재해싱 기반)
- **ILMV**: 양방향 상호 계층 검증
- **LPBFT**: 긴급 경량 PBFT 합의 프로토콜
- **BIVM**: 잔액 불변성 검증 모듈 (잔액 매핑 무결성 BMI 포함)
- **5-TVP**: 5단계 거래 검증 파이프라인
- PoW·PoS 없이 데이터 무결성 확률적 보장
- 오프라인 잠금(Offline Lock) 메커니즘
- 해시 검증 평균: **4.2ms**, 사실 확인: **1초 이내**

# PDV — Personal Data Vault
- 기록 원칙: **6하원칙** (누가·언제·어디서·무엇을·어떻게·왜)
- 저장: OpenHash 분산원장 불변 앵커링 (SHA-256 해시)
- 열람: **본인만** — 기관·정부·Gopang도 접근 불가
- Gate-Keeper: AI 비서(PAA)가 유일한 관문. 사용자 동의 없이 외부 전달 금지.
- 인증: IPv6 기반 고팡 SSO (`user.ipv6`)
- 증거 패키지 JSON(94): PAA가 생성, OpenHash 해시 일치 여부 포함
- 활용: K-Law 증거 제출 / K-Police 사건 기록 / K-Health 진료 기록 / PDV 로그

# K-Security (security.hondi.net)
- 아키텍처: **2단계 LLM 파이프라인**
  - 1단계: DeepSeek V3 스크리닝 (이상 점수 산출)
  - 2단계: 이상 점수 ≥ 0.6 → Claude Opus 심층 분석
- 심각도 3등급: S1(정보) / S2(경고) / S3(긴급)
- 보고 형식: 6하원칙 JSON (security-agent.js)
- Supabase 스키마: security_log · security_event · security_command
- 모니터링 주기: 24시간 간격 (폴링, 지수 백오프)

# Gopang 메인 앱 (gopang_v2)
- GWP(Gopang Widget Protocol): 서브시스템 간 연동. iframe 방식 → 새 탭 모드
- LLM 라우터: 키워드 매칭(gwp-registry.js) + DeepSeek LLM 라우터
- 기본 모델: DeepSeek V4 Flash / K-Law·K-Health는 V4 Pro
- SP 길이 제한: 10,000자 (토큰 비용 제어)
- GPS 권한 처리 포함
- 402 에러 메시지 처리 (BYOK 안내)

## OpenHash 기반 적응형 계층 구조 시스템 특허명세서 (원본 명세서)
- 핵심 개념: 블록체인의 "포괄적·무차별적 신뢰도"를 "제한적·차별적 신뢰도"로 대체 → 에너지 문제 해결
- PoW·PoS 등 합의 메커니즘 **전혀 없음** — 이것이 블록체인과의 본질적 차이
- 기록 시점 참(True) 보장 없음. **최초 기록 이후 위변조 미발생**을 신뢰도 범위 내 보장
- 확장성: 수학적으로 무제한 TPS (비트코인 7 TPS·이더리움 15 TPS 대비)
- 51% 공격 논점 → **개인 단말 보안 문제**로 전환
- 핵심 기능 4가지: 확률적 계층 선택 / 신뢰도 선택 / 실시간 검증·치유 / 해시 전용 전송(프라이버시)
- 해외 유사 연구 실패 사례 분석 포함: MIT 체인앵커 프로젝트, 스탠포드 샤딩 연구

## IPv6 농업 + OpenHash 통합 특허출원서
- 발명명: OpenHash 기반 IPv6 농업 데이터 통합 관리 시스템
- IPv6 주소를 **경작지 식별자**로 활용 (2^128개 주소 → 전국 2,847,000개 경작지 고유 할당)
- OpenHash로 농업 데이터 무결성 보장 + 농업인 데이터 주권 실현
- 기존 블록체인 농업 적용 실패 이유: 에너지 과소비(연 121 TWh) + 7 TPS 확장성 한계
- 종래 중앙집중식 문제 해결: 단일 장애점 제거 / 데이터 소유권 농업인 귀속 / 위변조 방지
