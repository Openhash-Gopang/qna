## 0. 전략 요약

고팡(Gopang)은 카카오톡과 유사한 AI 메신저이지만, 결정적으로 다른 점이 하나 있다.  
**카카오톡은 친구가 없으면 쓸모없지만, 고팡은 나 혼자서도 즉시 유용하다.**

K-Law로 법률 판결을 예측하고, K-119로 응급상황을 보고하며, K-Insurance로 보험료를 산출하고, K-Democracy로 시민 입법에 참여한다. 이 모든 것이 월 평균 100원의 비용으로 작동한다. 확산 전략의 출발점은 이 "1인 유용성"이다.

### 1-1. K-Law — 판결 시뮬레이터 (`klaw.hondi.net`)

**무엇을 하는가**  
사건의 핵심 사실관계를 입력하면 AI가 대법원 판결을 예측한다. K-Law v15.1 방법론을 적용하며, STEP 0(사건 분석·명제 설정) → STEP A(판례 법리 분석) → STEP B(법률관계 확정·판결문) → STEP C(확신도·자기검증) 4단계로 판결문을 생성한다.

**기술 스펙**
- AI 엔진: DeepSeek V4 Pro(무료 내장 프록시) / Claude Sonnet 4 / Gemini 2.5 Pro 선택
- 사건 분류: 민사(TYPE_01) · 형사(TYPE_02) · 행정(TYPE_03) · 가사(TYPE_04) · 소액(TYPE_05) · 노동(TYPE_06)
- 출력 지표: 법리 확신도/사실 확신도/종합 확신도(각 X/10), 사건 복잡도(X/10), 대법원 판결 일치도 예상(%)
- 결론 유형: 확정적 / 조건부 / 판단유보
- 데이터 저장: Supabase `klaw_cases` + `klaw_sessions` (판결문, 신뢰도, 사용자 이력 전량 기록)
- 인증: 고팡 SSO → `user.ipv6`로 PDV 자동 보고

**사업화 포인트**
- DeepSeek 무료 플랜으로 운용 중 → Claude·Gemini API Key는 사용자가 직접 입력하는 BYOK(Bring Your Own Key) 모델
- `klaw_sessions` 누적 데이터 → 사건 유형별 AI 정확도 통계 → 법무법인 B2B 세일즈 근거
- "허위 판례 번호 인용 금지" 가드레일 → 신뢰도 기반 프리미엄 플랜 전환 근거

### 1-2. K-119 — AI 응급출동 서비스 (`911.gopang.net`)

**무엇을 하는가**  
"AI 구조대원이 24시간 당신 곁에 있습니다." 화재·구조·구급·재난 모든 상황에 즉시 대응하는 AI 응급출동 서비스. 고팡 계정 하나로 로그인하면 전담 AI 구조대원이 즉시 배정되고 24시간 생명 감시가 시작된다. 제주도 시범 서비스.

**핵심 성능 지표**

| 지표 | 수치 |
|---|---|
| 평균 현장 도착 시간 | 5.8분 |
| AI 구조대원 대기 | 24시간 상시 |
| 중증도 분류 단계 | 4단계 (1급~4급) |
| 출동번호 형식 | FD-YYYYMMDD-NNNN (예: FD-20260604-2847) |

**대응 재난 유형 (전체)**

| 분류 | 세부 유형 |
|---|---|
| 화재 | 건물·산불·차량·가스화재 |
| 구조 | 건물붕괴·산악·엘리베이터·붕괴·고립 |
| 구급 | 심정지·뇌졸중·외상·중독 |
| 교통 | 교통사고·차량·오토바이·보행자 |
| 수난 | 익수·선박사고·해양재난 |
| 화학·특수 | 누출·폭발·CBRN·독성 |
| 자연재해 | 홍수·태풍·지진·폭설 |
| 기타 | 자살·정신응급·동물피해·기타 응급 |

**3계층 시스템 아키텍처**

| 계층 | 구성 | 역할 |
|---|---|---|
| Layer 1 — 시민 인터페이스 | K-119 앱(webapp.html) | 24시간 AI 채팅, GPS 실시간 위치, 중증도 자동 분류, 출동번호 발급, 응급처치 가이드 (음성·텍스트 동시 지원) |
| Layer 2 — AI 지휘 엔진 | SeverityEngine · DispatchEngine · HospitalEngine (서버리스) | 4단계 중증도 분류, Haversine 최적 배치, 1급 즉시 자동 출동, WebSocket 차량 지휘 |
| Layer 3 — 종합상황실 | dashboard.html | Canvas 실시간 출동 지도, 6뷰 통합 대시보드, 출동 승인·오버라이드, AV 자율주행 지휘, 병원 수용 조정 |

**4단계 중증도 분류 → 자동 대응**

```
1급 (위독)  →  즉시 자동 출동 · Haversine 최근접 소방서·구급대 자동 선발
2급 (중증)  →  출동번호 발급 후 배치
3급 (중등)  →  AI 대기 + 상황 모니터링
4급 (경증)  →  응급처치 가이드 제공 · 보건소 연결 권고
응급 감지    →  GPS·심박·낙상 센서 종합 분석 → 심정지·실신·낙상 자동 신고
```

**예방적 사전 감지 (웨어러블 연동)**  
GPS·심박·낙상 센서 데이터를 실시간 종합 분석 → 이상 징후 감지 시 자동 119 신고. 신호 없이도 선제적 출동 가능.

**응급처치 실시간 가이드**  
신고자에게 앰뷸런스 도착 전까지 CPR·지혈·기도 확보 방법을 단계별로 안내. 음성·텍스트 동시 지원.

**병원 자동 연결 흐름**
```
중증도 + 위치 → 수용 가능 응급실 자동 매칭
              → 병원 도착 전 환자 정보(PDV) 사전 전송
              → 병원 수용 여부 실시간 확인
              → WebSocket으로 구급차·소방차·헬기·AV 이송차량에 목적지·임무 직달
```

**PDV 6하원칙 출동 기록 샘플**

| 필드 | 예시 |
|---|---|
| who.ipv6 | 2601:db80:bd05:…… |
| what.severity | 1급 (위독) |
| what.dispatch_no | FD-20260604-2847 |
| what.arrival_min | 5.2분 |
| what.hospital | 제주대학교병원 |
| svc | '911' |

모든 출동·처치·이송·결과는 PDV 암호화 기록 → **사후 분석·법적 증거로 활용 가능**. NDMS(국가재난안전시스템) 연동.

**사람 소방관의 역할 보존**  
AI 시스템이 전 과정을 자동 처리하되, 사람 소방관·구급대원은 종합상황실에서 **출동 승인·오버라이드 권한**을 항시 보유. AI 독단 배제 구조.

**기술 스펙**
- AI: DeepSeek V4 Pro — 시스템 프롬프트: "화재·구조·구급·재난 상황에 즉각 대응, 응급처치 가이드 제공, 필요 시 119 자동 신고, 간결·정확"
- 프록시: `gopang-proxy.tensor-city.workers.dev` Cloudflare Worker v4.3
- 역지오코딩: Kakao Maps API
- DB: Supabase `pdv_log` (`svc: '911'`)
- 연동: K-Police · K-Law · K-Health · 고팡 마켓
- 인증: 고팡 SSO (`user.ipv6`) — `user.guid` 버그 수정 완료

**사업화 포인트**
- 소방청 공식 연동 → 국내 최초 AI 119 접수 시스템 → 행안부·소방청 B2G R&D 과제
- CBRN(화학·생물·방사능·핵) 대응 포함 → 국방부·행안부 특수 재난 연계
- 웨어러블 연동 예방적 감지 → IoT 스마트시티 인프라 수요 (제주도 스마트시티 예산 연계)
- NDMS 연동 → 전국 소방서 데이터 통합 → 재난 빅데이터 연구기관(KIST·ETRI) 협력
- AV(자율주행) 이송차량 WebSocket 지휘 → 자동차 OEM·자율주행 기업 API 공급
- 법적 증거력 있는 PDV 출동 기록 → 보험사 사고 처리 자동화 연동 (K-Insurance 시너지)

### 1-3. K-Police — AI 치안 서비스 (`police.gopang.net`)

**무엇을 하는가**  
"당신만을 위한 AI 경찰관이 24시간 지킵니다." 모든 시민·기관·사물에 전담 AI 경찰관이 배정된다. 별도 신청 없이 고팡 계정만으로 즉시 이용 가능. 범죄 예방 → 신고 접수 → 출동 지시 → 형사소송 개시까지 원스톱. 제주도 시범 서비스.

**핵심 성능 지표**

| 지표 | 수치 |
|---|---|
| 평균 출동 시간 | 4.2분 |
| AI 위험도 분류 속도 | 0.1초 |
| K-Law 국제 연동 | 193개국 |
| AI 경찰관 대기 | 24시간 상시 |

**대응 범죄 유형 (전체)**

| 분류 | 세부 유형 |
|---|---|
| 폭력 | 폭행·협박, 신체적 위협·공갈 |
| 재산 | 절도·강도, 재산 침해·강제 탈취 |
| 금융사기 | 보이스피싱 (전화·문자·기관 사칭), 투자사기·계좌 탈취 |
| 사이버 | 해킹·랜섬웨어·개인정보 침해 |
| 불법촬영 | 몰카·딥페이크·유포 |
| 스토킹 | 추적·미행·반복 접근 |
| 가정폭력 | 가정 내 학대·폭행 |

**3계층 시스템 아키텍처**

| 계층 | 구성 | 역할 |
|---|---|---|
| Layer 1 — 시민 인터페이스 | K-Police 앱(webapp.html) | 24시간 AI 채팅, GPS 위치 스트림, 일정 위험 예보, 범죄 신고 접수·사건번호 발급, 형사소송 패키지 생성 |
| Layer 2 — AI 지휘 엔진 | RiskEngine · DispatchEngine · LawsuitEngine (서버리스) | 위험 4단계 분류(0.1초), Haversine 자원 배치, CRITICAL 자동 출동, WebSocket AV 명령, BroadcastChannel 상황실 연동 |
| Layer 3 — 상황실(Ops) | dashboard.html | Canvas 실시간 사건 지도, 사건·요원·차량 통계, 출동 승인·오버라이드, AV 자율주행 명령 모니터링, 시뮬레이션 4종 |

**4단계 위험도 분류 → 자동 대응**

```
CRITICAL  →  즉시 자동 출동 · Haversine 최근접 요원 자동 선발
HIGH      →  출동 지시 + 상황실 알림
LOW/MED   →  AI 상담 + 예방 가이드 제공
예방 감지   →  GPS·일정·채팅 종합 분석 → 야간 이동·낯선 차량 탑승·우범 지역 접근 시 자동 경보
```

**예방적 치안 (사전 개입)**  
로그인 즉시 GPS·일정 모니터링 시작. 야간 이동, 낯선 차량 탑승, 해외 우범 지역 접근 감지 시 **위험 발생 전** 자동 경보 발령.

**형사소송 패키지 자동 생성**  
신고 대화에서 범죄 유형 자동 특정 → 국가(검찰청) vs 피의자 **형사소송 패키지 즉시 생성** → K-Law 법률 시스템 자동 연동. 시민이 변호사 없이도 소송 절차를 시작할 수 있는 구조.

**K-Law 국제 연동: 193개국**  
국내 범죄뿐 아니라 해외 우범 지역 접근 경보, 국제 사기·해킹 사건에 193개국 K-Law 법률 DB 자동 연동.

**PDV 6하원칙 기록 샘플**

| 필드 | 예시 |

| who.ipv6 | 2601:db80:bd05:…… |
| what.summary | K-Police 사건 접수 |
| svc | 'kpolice' (alias → 'police') |

모든 상담·사건·출동·소송 이력 PDV 암호화 기록 → **법적 증거로 직접 활용 가능**.

**자율주행 차량 지휘**  
WebSocket으로 AV 이송차량에 목적지·임무 직달 → 용의자 이송·현장 출동 완전 자동화.

**사람 경찰관의 역할 보존**  
전 과정 AI 자동 처리, 사람 경찰관은 상황실 대시보드에서 **출동 승인·오버라이드 권한** 보유. AI 독단 배제.

**기술 스펙**
- AI: DeepSeek V4 Pro — "24시간 보디가드, 범죄 예방·신고·소송·출동, 간결·정확"
- 프록시: `gopang-proxy.tensor-city.workers.dev` Cloudflare Worker v4.3
- 역지오코딩: Kakao Maps API
- DB: Supabase `ebbecjfrwaswbdybbgiu` `pdv_log` (`svc: 'kpolice'`)
- 연동: K-Law · K-119 · 고팡 마켓 · 고팡 세무(`tax.gopang.net`)

**사업화 포인트**
- 경찰청 공식 연동 → 국내 최초 AI 112 접수 시스템 → 행안부·경찰청 B2G R&D

### 1-4. K-Traffic — AI 교통·물류 플랫폼 (`traffic.gopang.net`)

**무엇을 하는가**  
우버·카카오택시와 유사하지만 신원·기록·결제의 방식이 근본적으로 다르다. 제주도 전체 차량을 실시간 연계하여 **동선 겹침 알고리즘**으로 최적 매칭하고, 전 과정을 양측 PDV에 이중 기록한다. 사람 이동과 화물 배송을 동일 플랫폼에서 처리한다.

**핵심 수치**

| 지표 | 값 |
|---|---|
| 운영 시간 | 24시간 365일 |
| 평균 매칭 시간 | 3분 |
| 신원 인증 | 100% PDV 기반 |
| 결제 | GDC 완료 즉시 자동 이체 |
| 기본 요금 | 2 GDC (5km 이내) + 0.3 GDC/km |

**Openhash 5계층 차량 추적 구조**

```
L1 (읍면동)  →  L2 (시군구)  →  L3 (광역시도)  →  L4 (국가)  →  L5 (글로벌)
```
- 모든 차량이 **1분 단위**로 위치·동선을 Openhash L1 서버에 전송 (블랙박스)
- 탐색 원칙: L1 이내 동선이면 L2로 전달하지 않음 → 프라이버시 최소 노출
- PDV L1 미만 차량은 매칭 추천에서 자동 제외

**7단계 AI 교통 워크플로**

```
1. 이동 요청   →  출발지·목적지·인원·시간 입력 · PDV 이력 참조 경로 자동 추천
2. 차량 탐색   →  동선 겹침 + PDV L1 이상 + 적재 가능 차량 선정
3. 릴레이 조합 →  장거리는 환승 거점(공항·항·터미널·물류센터)에서 복수 차량 조합
4. 요금 사전고지 →  확정 요금제 · GDC 잔액 부족 시 탑승 전 안내
5. 운전자 수락  →  자동 요청 전송 · 거절 시 다음 최적 차량 재탐색
6. PDV 신원확인 →  이름·차량번호·PDV 신뢰 점수 상호 표시 · 미인증 차량 탑승 불가
7. GDC 정산·평가 →  도착 즉시 자동 이체 · 상호 평가 PDV 저장 · 영구 기록
```

**릴레이 인수인계 메시지 형식** (AI 자동 생성)
```
📍 수령 장소 / 📦 수령 물품 / 🏁 전달 장소
🔄 인수인계 / ⏰ 시간 / 💰 GDC / 🔐 PDV 기록
```

**주요 환승 거점 (제주 파일럿)**  
제주국제공항 · 제주항 · 한림항 · 제주도청 · 서귀포시청

**기존 서비스 vs K-Traffic**

| 비교 항목 | 우버·카카오택시 | K-Traffic |
|---|---|---|
| 차량 탐색 | 반경 내 임의 차량 | 동선 겹침 알고리즘 최적 매칭 |
| 신원 확인 | 휴대폰 번호만 | 양측 PDV 기반 상호 확인 |
| 거래 기록 | 플랫폼 서버 저장 | 양측 PDV 암호화 — 본인만 열람 |
| 결제 | 카드·현금·포인트 | GDC 완료 즉시 자동 이체 |
| 분쟁 해결 | 플랫폼 중재 (수일) | PDV 기록이 즉시 증거 |
| 범죄 예방 | 사후 신고 | 전 과정 기록 → 구조적 억제 |
| 차량 종류 | 택시 또는 자가용 | 버스·택시·자가용·화물차 통합 |
| 물류 | 별도 앱 | 동일 플랫폼 화물·배송 통합 |

**기술 스펙**

- 우버·카카오택시 대비 **PDV 기반 범죄 구조적 억제** → 여성·야간 안전 이동 수요 직접 공략

### 1-7. K-Tax — AI 세무 자동화 (`tax.gopang.net`)

**무엇을 하는가**  
"단 1원의 세금도 더 내지도, 덜 내지도 않습니다." K-Tax는 고팡 플랫폼의 세무 자동화 시스템. 시민과 기업의 거래 원장(fs_ledger)에서 데이터를 자동 수집하고, 각국 현행 세법을 즉시 적용하며, 계산된 세금을 실시간으로 정부 계좌에 이체한다. GDC를 결제 수단으로 사용하는 거래는 **탈세·불법·편법·위법이 구조적으로 불가능**하다.

**5단계 자동화 파이프라인 (SP-TAX v2.0)**

```
Step 1 — 데이터 수집   : market 거래 원장(fs_ledger)에서 revenue·opex 실시간 집계
Step 2 — 세법 적용     : 현행 한국 세법 — 부가세·종합소득세·지방소득세 자동 계산
Step 3 — 납부 실행     : 사용자 확인 후 납세 트랜잭션 fs_ledger 기록 · 정부 계좌 자동 이체
Step 4 — 보고서 저장   : tax_reports 테이블에 JSONB 전문 보고서 저장 (Supabase)
Step 5 — PDV 기록      : 6하원칙으로 납세 이력 PDV 원장 영구 기록 · OpenHash 불변 앵커링
```

**AI 세무 비서**  
자연어로 세금을 물어보면 5단계 파이프라인이 자동 실행된다. 사람의 개입이 없으며 오직 시스템에 의해 동작.

### 1-8. GDC — Gopang Digital Currency (`gdc.gopang.net`)

**무엇을 하는가**  
"5천만 인구와 5백만 기관이 구성하는 단 하나의 결합 재무제표." GDC(Global Digital Currency)는 193개국 국적 통화의 POOL이자, 고팡 생태계 전체의 결제·금융 인프라다. 사전 채굴이나 중앙 발행 없이, 사용자의 실제 입금에 의해서만 발행된다. 모든 메커니즘과 소스코드는 오픈소스.

**GDC 발행 원리 (사전 채굴 없음)**

```
사용자 KRW 10,000원 입금
  → 교환비(₮1 = KRW 1,000) 적용
  → GDC 10₮ 즉시 발행 → 사용자 계좌 귀속
  → POOL 잔고 += KRW 10,000
  → 각국 Index 증권 즉시 투자
런칭 시점 POOL 잔고: 0원 (완전 투명)
```

**6개 지원 통화 (FIAT POOL)**

| 통화 | 기준 |
|---|---|
| KRW 대한민국 원 | ₮1 = KRW 1,000 (기준가) |
| USD 미국 달러 | 실시간 환율 |
| EUR 유럽 유로 | 실시간 환율 |
| JPY 일본 엔 | 실시간 환율 |
| CNY 중국 위안 | 실시간 환율 |
| GBP 영국 파운드 | 실시간 환율 |

환전 수수료 0% · 0.1초 이내 완료 · 24시간 365일

**핵심 금융 서비스 5종**

### 1-9. K-Market — AI 자율 시장 플랫폼 (`market.gopang.net`)

**무엇을 하는가**  
"소비자 데이터 수집 0%. 오직 사용자의 이익을 위해 복무하는 AI 비서." K-Market는 고팡 생태계의 AI 자율 시장 플랫폼(v1.0). 기존 이커머스(쿠팡·네이버쇼핑)가 소비자 구매 이력을 전면 수집하는 것과 달리, K-Market는 소비자 데이터를 **0%** 수집하며 AI Agent가 오직 해당 사용자의 이익만을 위해 복무한다. 모든 거래는 GDC로 결제되고 K-Tax로 세금이 자동 처리된다.

**핵심 성능 지표**

| 지표 | 수치 |
|---|---|
| 거래 처리 속도 | 7.3M TPS |
| AI 수요 예측 정확도 | 94.7% |
| 소비자 데이터 수집 | 0% |
| 전력 절감 | 99.8% |
| 지원 범위 | GPID 글로벌 단일 시장 (193개국) |

**AI Agent의 역할 — 구매자 측**

```
1. 구매 필요성 검토  →  "이 상품이 정말 필요한지" AI가 먼저 검토·의견 제시
2. 최적 상품 탐색   →  품질·가격 최우수 상품 자동 선정
3. 가격 비교        →  전 세계 동일 상품 GPID 기준 실시간 비교
4. 주문·결제 처리   →  GDC 자동 결제 · OpenHash 거래 증빙 불변 기록
5. 재무제표 반영    →  모든 거래 → 사용자 재무제표·PDV 자동 갱신
```
비유: AI 공정거래위원회가 24시간/365일 동행 → 사기·바가지 구조적 불가.

**AI Agent의 역할 — 판매자 측**

```
1. 사업성 검토      →  창업 전 수요 예측·경쟁 분석·최종 의견 제시
2. 웹사이트 자동 생성 →  상품 설명 서술 한 마디 → DeepSeek V4 Pro가 완성형 판매 사이트 생성
3. 경영 전략 수립   →  일간·주간·월간·분기 업무 계획 자동 제시 및 갱신
4. 재고·물류 자동화 →  부족 예측 → 자동 발주 → K-Logistics 연동 → 배송 추적
5. 세무 자동화      →  재무제표 자동 생성·갱신 + K-Tax 연동 세금 자동 납부
6. 인사 관리        →  직원 복지·보상 계획 수립 및 집행
```

**판매자 사이트 자동 생성 (DeepSeek V4 Pro)**

### 2-4. 바이럴 레이어 — AI 결과물이 광고가 된다

카카오톡은 대화를 공유한다. 고팡의 바이럴 단위는 **AI가 생성한 결과물**이다.

- K-Law 판결문 하단: `"K-Law v15.1 · Gopang AI가 작성 — klaw.hondi.net"`
- K-Insurance 보험료 비교표 하단: `"K-Insurance · gopang.net/insurance"`
- K-Democracy 법안 초안 하단: `"K-Democracy AI 숙의 파이프라인 · democracy.gopang.net"`

결과물이 카카오톡·이메일·SNS로 공유될 때마다 자동 광고가 된다.  
공유율(K 계수) 1건당 평균 3명 노출 가정 시, 월 1,000건 판결문 = 3,000명 노출.

### 2-5. 락인 구조 — PDV가 이탈 방지 장치다

카카오톡은 친구 목록이 락인이다.  
고팡의 락인은 **PDV에 쌓이는 나의 AI 이력**이다.

- K-Law: 내가 의뢰한 사건의 법리 분석 이력 → `klaw_sessions`
- K-119: 나의 응급 위치·신고 이력 → `pdv_log`
- K-Insurance: 나의 보험 계약·청구 이력 → `insurance_calc_results`
- K-Democracy: 나의 입법 참여 이력

이 데이터는 OpenHash 분산원장에 앵커링되어 **다른 앱으로 이전 불가능**하다.  
서비스를 오래 쓸수록 AI 비서의 정확도가 높아지는 구조.

## 8. 결론

고팡의 사업화 핵심은 **이미 작동하는 14개 서브시스템**이다.

- **K-Law**: 대한민국 어디서나 즉시 체험 — 최초 접점
- **K-119**: 소방청 연동 시 국내 최초 AI 119 — B2G 레퍼런스
- **K-Police**: 경찰청 연동, 형사소송 자동화, 193개국 연동 — AI 치안
- **K-Health**: 전 생애 PDV + PubMed 3,700만 편 — AI 평행 병원
- **K-Insurance**: GDC(₮) 기반 — 제주도 폐쇄 경제 실험의 핵심
- **K-Traffic**: 동선 겹침 매칭 + PDV 이중 기록 — 우버·카카오택시 대체재
- **K-Logistics**: 1~3차 산업 공급망 단일 통합 — 물류비 34% 절감
- **K-Tax**: GDC 기반 탈세 구조적 불가, 8단계 누진세 자동화 — 국민 세무
- **GDC**: 193개국 FIAT POOL · 수수료 0% · 재무제표 0.1초 신용평가 — 글로벌 금융
- **K-Market**: 소비자 데이터 0% · GPID 글로벌 단일 상품 ID · AI 자율 거래 — AI 마켓플레이스
- **K-School**: 평생 1:1 AI 교수 · 166개 과목 · 26개 언어 · 50년 진로 예측 — AI 교육
- **K-Public**: 말 한 마디 민원 처리 · 89% 시간 단축 · 행정 부패 구조적 방지 — AI 정부
- **K-Stock**: 89개 자산군 · 기존 PB 1/10 비용 · 연 148만 원 절세 — AI 자산관리
- **K-Democracy**: DAWN 철학 + 이원 권력 구조 — 법률 풀스택 완성

이 9개가 하나의 앱(고팡)으로 묶이는 순간, 경쟁자가 없다.  
변호사·소방관·경찰관·의사·보험 설계사·택시기사·물류 관제원·세무사·은행원·자산관리사·마켓 운영자·교사·공무원·국회의원의 역할을  
AI가 부분 대체하면서도 사용자 데이터는 PDV에만 저장된다.

**제주도 안에서 완성되는 11개의 풀스택:**

1. **법률 풀스택** — K-Law(판결 예측) + K-Police(형사소송) + K-Democracy(입법)
