## 0. 핵심 목표 재정의

단순 기술 검증이 아니다.

> **"제주도민과 관광객이 고팡 없이는 불편하다고 느끼는 임계점(N)을 넘는 것"**

| 임계점 지표 | 목표치 |
|---|---|
| 일일 활성 사용자 (DAU) | **2,000명** |
| 주간 거래액 | **1,000 GDC 이상** |

제주도라는 폐쇄된 경제·행정·관광 생태계 안에서 네트워크 효과의 최소 임계점을 먼저 달성하는 것이 전략의 핵심이다.

## 1. 왜 제주도인가

| 조건 | 제주도 현황 | 고팡 연결 |
|---|---|---|
| 인구 규모 | 약 70만 명 | 폐쇄 경제 실험에 적합한 최소 임계 규모 |
| 관광객 | 연 1,500만 명 | GDC 거래량·K-Traffic 수요 즉시 확보 |
| 스마트시티 예산 | 약 20억 원/년 | B2G 진입 명분 및 과제 편성 요청 가능 |
| 기존 파일럿 | K-119·K-Police·K-Insurance 이미 제주 시범 운용 중 | 추가 비용 없이 확장 가능 |
| 지리적 특성 | 섬 = 완결된 행정·교통·물류 단위 | 전 서비스 연동 효과 즉시 측정 가능 |
| GDC 기준가 | ₮1 = KRW 1,000 제주 파일럿 기준 설정 완료 | CBDC 연구 사례로 한국은행 협력 가능 |

## 4. 기술·운영 고려사항

### 4-1. 오프라인 대응 (네트워크 음영 지역)

제주 산간·해안(한라산 등반로·우도·마라도)은 LTE/5G 불안정.

- K-119 신고 메시지: **SMS 폴백** + 위치 좌표 로컬 저장 후 재전송 로직 구현
- PDV 동기화: Wi-Fi 연결 시 **배치 처리**

### 4-2. 다국어 지원 (외국인 관광객)

제주 방문 외국인(중국·일본·동남아) 대응:

- K-119 · K-Traffic: **영어·중국어·일본어 UI** 우선 제공
- AI 응답: DeepSeek 프록시 시스템 프롬프트에 "답변은 사용자 설정 언어로" 포함

### 4-3. GDC 법적 안전장치

- 현재: **포인트·마일리지 성격**으로 운영
- 이자 지급 표현: "적립금" 명목
- 환전 표현: "포인트 교환"
- **3개월 내**: 금융위원회 혁신금융서비스 샌드박스 신청
- POOL 은행 계좌에 **5천만 원 예치** → 24시간 자동 환전 봇 운영

### 4-4. K-119 오출동 방지

1급 자동 출동 전 AI "정말 119가 필요한가요? 예/아니오" **2회 음성 확인** 단계 추가.
